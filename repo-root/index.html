üõ†Ô∏è Implementation Steps

1

üñ•Ô∏è Environment Setup - EC2 Instance

Created and configured an Amazon EC2 instance to host our web application.

  • Launched EC2 instance with Amazon Linux 2 AMI
  • Configured Security Group to allow HTTP (port 80) and SSH (port 22)
  • Installed Apache web server (httpd) for serving web pages
  • Installed CodeDeploy Agent for automated deployments
  • Tagged instance with Name=CodeDeploy for identification
sudo yum install -y httpd ruby wget
sudo systemctl start httpd && sudo systemctl enable httpd
2

üîê IAM Roles & Policies

Created necessary IAM roles for secure service-to-service communication.

  • EC2CodeDeployRole - Allows EC2 to communicate with CodeDeploy and S3
  • CodeDeployServiceRole - Allows CodeDeploy to manage EC2 instances
  • CodePipelineServiceRole - Allows Pipeline to access CodeCommit and CodeDeploy
  • Attached policies: AmazonEC2RoleforAWSCodeDeploy, AmazonSSMManagedInstanceCore, AWSCodeDeployRole
3

üì¶ CodeCommit Repository

Set up AWS CodeCommit as our Git repository for version control.

  • Created repository: TCS-ION-DevOps-App
  • Configured Git credentials helper for authentication
  • Pushed application code including index.html, appspec.yml, and deployment scripts
  • Branch: main
git remote add origin https://git-codecommit.us-east-1.amazonaws.com/v1/repos/TCS-ION-DevOps-App
git push -u origin main
4

üöÄ CodeDeploy Application

Configured CodeDeploy to handle the deployment process to EC2.

  • Created Application: TCS-ION-DevOps-App
  • Compute Platform: EC2/On-Premises
  • Created Deployment Group: TCS_ION_DevOps_DG
  • Deployment Type: In-place deployment
  • Deployment Config: CodeDeployDefault.OneAtATime
  • Uses appspec.yml to define deployment lifecycle hooks
5

üîÑ CodePipeline Setup

Created the pipeline to automate the entire CI/CD workflow.

  • Pipeline Name: TCS-ION-DevOps-Pipeline
  • Source Stage: AWS CodeCommit (triggers on push to main branch)
  • Deploy Stage: AWS CodeDeploy (deploys to EC2)
  • Artifact Store: S3 bucket for storing deployment packages
  • Automatic execution on every git push
6

üìù AppSpec Configuration

Created appspec.yml to define how CodeDeploy should deploy the application.

  • Specifies file destinations (source ‚Üí /var/www/html)
  • ApplicationStop hook: Runs stop.sh script
  • AfterInstall hook: Runs after_install.sh to restart Apache
version: 0.0
os: linux
files:
- source: /
destination: /var/www/html

‚ö†Ô∏è Challenges & Solutions

Real problems faced during implementation and how they were resolved

üêß

Service Naming Conflict

Problem: Initial deployments failed because the script referenced apache2 (Ubuntu), but the server was Amazon Linux 2.

Solution: Analyzed deployment logs, identified OS difference, and updated scripts to use httpd (Amazon Linux).

üåç

Region Mismatch

Problem: Resources were being created in different AWS regions (ap-south-1 vs us-east-1), causing connectivity issues.

Solution: Standardized all resources to us-east-1 region and verified AWS CLI default configuration.

üîê

IAM Permission Errors

Problem: CodeDeploy agent couldn't pull artifacts from S3 due to missing permissions on EC2 instance.

Solution: Created proper IAM Instance Profile with AmazonS3ReadOnlyAccess and attached it to EC2.

üí° Key Learning: "Always check OS-specific service names and ensure all AWS resources are in the same region."
